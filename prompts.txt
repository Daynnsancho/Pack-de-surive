Crée-moi un site professionnel et moderne pour une boutique en ligne de sacs de survie entièrement personnalisables. Le concept est simple : l'utilisateur peut choisir ce qu'il veut mettre dans son sac (nourriture, outils, objets de survie). Chaque objet ajouté augmente le prix total et le poids total du sac, qui doivent s'afficher en temps réel. Le site doit être propre, sérieux, immersif, avec un design nature/survie (tons sombres, verts, oranges). Il doit contenir :  Une page d’accueil avec un slogan percutant  Une page pour configurer son sac de survie  Une section “objets disponibles” avec prix et poids  Une page contact / à propos  Fais en sorte que ce soit responsive (mobile + PC), rapide et facile à utiliser. Utilise HTML, CSS, JavaScript ou React (au choix). Fais en sorte que je puisse facilement le publier gratuitement (par exemple sur Vercel ou GitHub Pages).
Ajout dynamique des objets dans le sac :  Le script JavaScript mentionné dans le HTML n'est pas complet (la partie // Items will be added here by JavaScript est vide).  👉 Solution : Ajouter un tableau d’objets (nom, catégorie, poids, prix) en JavaScript avec fonctions pour afficher les items, filtrer par catégorie, et les ajouter au panier.  Calcul du prix et du poids total :  Il manque la logique de mise à jour dynamique de #total-weight et #total-price selon les objets ajoutés.  👉 Solution : Utiliser des événements JavaScript pour mettre à jour ces valeurs en temps réel.  Bouton “Finaliser la commande” :  Aucun traitement ou redirection n’est défini.  👉 Solution : Rediriger vers une page de paiement ou déclencher une modale de confirmation.  🎨 Design & UI Manque d’animations fluides :  Il pourrait y avoir des transitions plus subtiles entre les sections.  👉 Solution : Utiliser Tailwind avec Framer Motion ou ajouter plus d'effets transition / duration.  Accessibilité mobile :  Le menu mobile fonctionne, mais il n'est pas assez accessible (pas de focus trap, pas de fermeture avec "Escape").  👉 Solution : Ajouter des contrôles d'accessibilité ARIA + clavier.  Images optimisées :  Les images sont en très haute résolution sans loading="lazy" ou width/height.  👉 Solution : Ajouter loading="lazy" + spécifier les dimensions pour de meilleures performances.  Contraste de lecture :  Certaines zones de texte sur fond gris foncé peuvent manquer de contraste.  👉 Solution : Utiliser des couleurs plus contrastées ou éclaircir légèrement les zones de texte.  📱 Responsive Design Disposition du panier sur petit écran :  Sur mobile, le panier passe en dessous mais peut devenir difficile à faire défiler.  👉 Solution : Ajouter un max-h-[...] avec scroll auto + un bouton "Mon Sac" qui l’ouvre dans une modale sur mobile.  Disposition des boutons :  Certains boutons sont trop petits ou mal espacés sur mobile.  👉 Solution : Agrandir les zones tactiles (padding) pour meilleure ergonomie.  📈 Performance Optimisation du chargement :  Le site charge des polices d’icônes (FontAwesome) entières.  👉 Solution : Utiliser des SVG inline ou charger uniquement les icônes nécessaires.  Poids total des assets :  Image de fond très lourde (>1MB).  👉 Solution : Utiliser des versions compressées (WebP ou JPEG optimisé).  🛡️ Accessibilité & SEO Balises alt manquantes :  L’image de l’équipe dans la section “À propos” n’a pas de alt descriptif utile.  👉 Solution : Ajouter un texte significatif, ex. alt="Équipe Survival Pack en pleine nature".  Balises aria-* et navigation clavier :  Pas de support clair pour les lecteurs d’écran.  👉 Solution : Ajouter aria-label, role, et assurer la navigation complète via Tab.  Balise <form> sans action ni validation :  Le formulaire de contact est statique.  👉 Solution : Ajouter des validations JS + définir action (ou utiliser mailto: ou une intégration comme Formspree).  🧩 Fonctionnalités supplémentaires recommandées 🔍 Recherche rapide parmi les objets.  💾 Sauvegarde locale du sac avec localStorage.  📦 Préconfigurations de sacs selon des scénarios (rando, catastrophe, urbain...).  📊 Indicateur de poids max recommandé (pour avertir en cas de sac trop lourd).
